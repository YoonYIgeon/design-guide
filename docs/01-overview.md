# 01. 개요 (Overview)

## 목적

이 라이브러리는 **외부 사내 시스템의 관리자 페이지**를 구성하기 위한 재사용 가능한 UI 컴포넌트·디자인 규칙 모음입니다.
대상 시스템은 보안·운영상 이유로 **외부 네트워크와 철저히 분리**되어 있으며, 라이브러리는 **Git을 통해 배포·업데이트**됩니다.

## 범위 (Scope)

**포함**
- 관리자 화면 공통 UI 컴포넌트 (테이블, 폼, 모달, 내비게이션, 알림 등)
- 디자인 토큰 (색상, 타이포그래피, 간격, 반경, 그림자 등)
- 레이아웃 규칙 (관리자 셸, 사이드바, 헤더, 콘텐츠 영역)
- 접근성·상태 표현 규칙 (로딩/빈 상태/에러/권한)
- 위 요소를 쓰기 위한 가이드 문서

**미포함**
- 도메인별 비즈니스 로직 (각 소비 시스템이 소유)
- 서버/API 구현
- 인증·인가 정책 자체 (라이브러리는 UI 표현만 제공)

## 대상 사용자

| 역할 | 사용 방식 |
| --- | --- |
| 소비 시스템 개발자 | 라이브러리를 설치해 관리자 화면을 조립 |
| 라이브러리 유지보수자 | 컴포넌트·토큰·문서를 추가/수정하고 릴리스 |
| 디자이너 | 디자인 토큰·컴포넌트 스펙을 정의하고 리뷰 |

## 용어 정의

- **소비 시스템 (Consumer)**: 이 라이브러리를 가져다 쓰는 사내 관리자 애플리케이션.
- **격리 환경 (Isolated/Air-gapped Env)**: 공용 인터넷·레지스트리에 직접 접근할 수 없는 실행/빌드 환경.
- **내부 Git (Internal Git)**: 사내에서 접근 가능한 Git 호스팅(예: GitLab/Gitea/Bitbucket 온프레미스).
- **배포 아티팩트 (Artifact)**: 소비 시스템이 실제로 사용하는 산출물(소스, 번들, tarball 등).
- **디자인 토큰 (Design Token)**: 색/간격/타이포 등 디자인 결정을 코드화한 값.

## 설계 전제 (Assumptions)

문서 전반은 다음을 **권장 기본값**으로 가정합니다. 팀 결정에 따라 조정하세요.

- 프론트엔드 스택: **TypeScript + React** (프레임워크 무관 토큰은 별도 노출)
- 배포 경로: **내부 Git 저장소의 태그** 참조가 1차, tarball/사내 레지스트리 미러가 2차
- 버전 관리: **SemVer**
- 문서 언어: 한국어 (컴포넌트 API 명세는 영문 식별자 허용)
