# 07. 기여 가이드 (Contributing)

이 라이브러리는 여러 관리자 시스템의 UI 기반이므로, **일관성과 하위 호환**을 최우선으로 기여합니다.

## 기여 전 확인

- 변경이 **하위 호환**인지 판단([06 SemVer](06-versioning-release.md) 참고).
- 새 컴포넌트/토큰인지, 기존 확장인지 구분(중복 재발명 지양).
- 격리 환경 제약 준수: **외부 네트워크 호출·외부 CDN·비밀정보 금지**.

## 브랜치 & PR

- `main`에서 분기: `feature/*`, `fix/*`, `docs/*`, `chore/*`.
- PR 하나 = 논리적 변경 하나.
- PR 설명에 포함:
  - 변경 요약 / 배경
  - 파괴적 변경 여부 (`breaking` 라벨)
  - 스크린샷·GIF (시각 변경 시)
  - 마이그레이션 노트 (해당 시)
  - 문서/CHANGELOG 갱신 여부

## 커밋 컨벤션 (Conventional Commits)

```
<type>(<scope>): <설명>

feat(data-table): 컬럼 정렬 지원
fix(modal): ESC 키로 닫히지 않던 문제 수정
docs(ui): 접근성 규칙 보강
chore(build): 토큰 빌드 스크립트 정리
```

- **type**: `feat`, `fix`, `docs`, `style`, `refactor`, `test`, `chore`, `build`.
- 파괴적 변경은 본문에 `BREAKING CHANGE:` 명시.

## 코드 규칙

- 하드코딩 값 금지 → **디자인 토큰** 사용.
- 컴포넌트는 접근성(키보드·포커스·레이블) 기본 포함.
- 공개 API(props)는 타입·기본값·설명을 명확히. 변경 시 문서 동기화.
- 외부 의존성 추가는 리뷰에서 **보안·라이선스·격리 재현성**을 함께 검토.
- Lint/포맷은 저장소 설정을 따름(설정 추가 시 이 문서 갱신).

## 문서 규칙

- 컴포넌트/토큰 변경은 **같은 PR**에서 `docs/`와 `CHANGELOG.md` 갱신.
- 예제는 실제 export 이름과 일치시킴.
- "설계 전제/권장 기본값"이 팀 결정으로 확정되면 해당 문서를 갱신.

## 빌드 산출물(dist) 정책

> 확정: **A(dist 커밋 안 함)** — `dist/` 는 gitignore 되어 있습니다.
> 소비 시스템이 git 태그 의존성으로 설치하면 이 패키지의 `prepare` 스크립트(`yarn build`)가
> 설치 시점에 dist(번들 + 타입 선언)를 자동 생성합니다. ([03. 시작하기](03-getting-started.md))
> 빌드가 불가능한 완전 폐쇄망 소비자에게는 릴리스 시 tarball 아티팩트를 제공하세요.
