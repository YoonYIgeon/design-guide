# 04. UI 가이드라인 (Design & Components)

관리자 페이지의 일관성·가독성·유지보수성을 위한 규칙입니다.
모든 컴포넌트는 이 규칙과 디자인 토큰을 따릅니다.

## 디자인 원칙

1. **정보 밀도 우선**: 관리자 화면은 데이터가 많다. 여백보다 **스캔 가능성·정렬·정보 위계**를 우선.
2. **예측 가능성**: 같은 동작은 같은 위치·같은 컴포넌트로. 화면마다 재발명하지 않는다.
3. **상태를 숨기지 않는다**: 로딩/빈/에러/권한없음 상태를 항상 명시적으로 표현.
4. **되돌릴 수 있게**: 파괴적 동작(삭제 등)은 확인·되돌리기 경로 제공.
5. **접근성 기본 탑재**: 키보드 이동, 포커스 표시, 대비, 레이블은 옵션이 아니라 기본.

## 디자인 토큰

토큰은 하드코딩을 대체합니다. 컴포넌트·화면은 **토큰만 참조**합니다.

| 범주 | 예시 토큰 | 비고 |
| --- | --- | --- |
| 색상 | `--au-color-primary`, `--au-color-danger`, `--au-color-bg`, `--au-color-text` | 의미 기반 네이밍(용도) 우선 |
| 타이포 | `--au-font-size-sm/md/lg`, `--au-font-weight-*` | 시스템 폰트 스택 기본, 웹폰트는 번들 포함 |
| 간격 | `--au-space-1..8` (4px 스케일) | 마진/패딩은 스케일 값만 사용 |
| 반경 | `--au-radius-sm/md/lg` | |
| 그림자 | `--au-shadow-1/2/3` | 과도한 그림자 지양 |
| z-index | `--au-z-modal/toast/popover/alert` | 레이어 충돌 방지. 겹침 순서: modal < toast < popover < alert |

- **네이밍**: `--au-<범주>-<의미>` (예: `--au-color-danger`). 색상 raw 값(`--au-blue-500`)은 내부 팔레트로만.
- **다크모드/테마**: 토큰을 `:root`와 `[data-theme="dark"]`에서 재정의.

## 레이아웃 규칙

- **AdminShell**: 사이드바(내비) + 상단바(컨텍스트/사용자) + 콘텐츠 영역의 3분할.
- **콘텐츠 최대 폭**: 데이터 테이블 등은 전체 폭, 폼 등은 가독 폭(예: 720–960px) 제한.
- **간격 스케일**: 4px 배수만 사용.

## 컴포넌트 카탈로그 (기본 세트)

| 컴포넌트 | 용도 | 필수 상태 |
| --- | --- | --- |
| `Button` | 액션 | default/hover/active/disabled/loading |
| `DataTable` | 목록·정렬·페이지네이션 | loading/empty/error |
| `Form` / `Field` | 입력·검증 | error/hint/disabled/required |
| `Modal` / `Dialog` | 확인·상세 | 포커스 트랩·ESC 닫기 |
| `Toast` | 비침습 알림 | success/error/info |
| `Nav` / `Sidebar` | 내비게이션 | active/collapsed |
| `Badge` / `Tag` | 상태 표시 | 의미색 매핑 |
| `Pagination` | 페이지 이동 | |
| `EmptyState` | 빈 데이터 | |

## 상태 표현 규칙 (관리자 필수)

- **로딩**: 스켈레톤 또는 인라인 스피너. 버튼은 loading 시 중복 클릭 차단.
- **빈 상태**: `EmptyState`로 "무엇이 없는지 + 다음 행동" 제시.
- **에러**: 사용자 언어로 원인·재시도 경로 안내. 원시 에러 문자열 노출 금지.
- **권한 없음**: 요소 숨김/비활성 정책을 화면 간 일관되게.

## 접근성 (A11y)

- 모든 인터랙티브 요소는 **키보드 접근·포커스 링** 제공.
- 색상만으로 정보 전달 금지(아이콘/텍스트 병행).
- 텍스트 대비 최소 **WCAG AA (4.5:1)**.
- 폼 필드는 `label` 연결, 에러는 `aria-describedby`로 안내.
- 모달은 포커스 트랩 + 복귀 포커스.

## 카피(문구) 규칙

- 버튼은 동사 위주(예: "삭제", "저장"). 모호한 "확인" 남용 지양.
- 파괴적 동작 확인문은 대상·결과를 명시(예: "사용자 3명을 삭제할까요? 되돌릴 수 없습니다.").
- 관리자 화면은 **경어체 간결형**으로 통일.

## 하지 말아야 할 것

- 컴포넌트 안에서 데이터 패칭/상태/비즈니스 로직 수행 → **프레젠테이션 전용 원칙 위반** ([08-presentational-only.md](08-presentational-only.md)).
- 인라인 하드코딩 색/간격(토큰 우회).
- 외부 CDN 폰트/아이콘 로드(격리 위반).
- 화면별 일회성 버튼/모달 재구현.
- 접근성 속성 생략.
