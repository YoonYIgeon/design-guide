# 02. 아키텍처 (Architecture)

## 전체 그림

```
┌────────────────────────────┐        ┌─────────────────────────────┐
│  라이브러리 저장소          │  git   │  소비 시스템 (격리 환경)     │
│  design-guide              │ ─────▶ │  admin app A / B / C ...     │
│  - src/ (컴포넌트·토큰)     │  태그   │  - design-guide 를 의존성으로 │
│  - docs/ (가이드)          │  참조   │    참조 (git / 서브모듈 /     │
│  - 릴리스 태그 vX.Y.Z       │         │    사내 미러)                │
└────────────────────────────┘        └─────────────────────────────┘
          ▲
          │ PR / 리뷰 / 태깅
   유지보수자 · 디자이너
```

핵심: **소비 시스템은 라이브러리의 "특정 Git 태그"를 가리킨다.** 이 고정 참조가 재현성과 롤백을 보장합니다.

## 격리 환경에서의 배포 전략

격리 환경은 공용 npm/CDN에 접근하지 못할 수 있으므로, **아래 우선순위**로 배포 경로를 둡니다.

1. **내부 Git 태그 직접 참조 (권장 1차)**
   - `package.json`의 의존성으로 내부 Git URL + 태그를 지정.
   - 장점: 별도 인프라 없이 버전 고정·롤백 용이.
   - 예: `git+https://git.internal.company/design-guide.git#v1.4.0`

2. **Git Submodule**
   - 소비 시스템 레포에 서브모듈로 포함하고 특정 커밋에 고정.
   - 장점: 소스 레벨 통합, 오프라인 빌드에 강함. 단점: 서브모듈 운영 부담.

3. **사내 레지스트리/미러 (선택)**
   - 사내 Verdaccio/Nexus 등에 배포 아티팩트를 미러링.
   - 장점: 표준 패키지 워크플로우. 단점: 인프라 유지 비용.

4. **오프라인 tarball**
   - 완전 폐쇄망 대비. 릴리스마다 `yarn pack` 산출물(tgz)을 아티팩트로 첨부/보관.

> **원칙**: 어떤 경로든 **버전은 반드시 불변(immutable) 참조(태그/커밋 SHA/체크섬)**로 고정합니다. `main`이나 브랜치 HEAD를 직접 참조하지 않습니다.

## 의존성 관리 (격리 환경 재현성)

- 런타임 의존성은 **최소화**하고, 불가피하면 **peerDependencies**로 소비 시스템에 위임(예: `react`).
- 빌드 도구/개발 의존성은 격리 환경에서 재현 가능해야 하므로:
  - `package-lock.json`(또는 lockfile) **커밋 필수**.
  - 사내 미러가 있으면 `.npmrc`의 registry를 사내로 지정.
  - 필요 시 벤더링(사전 빌드 산출물 커밋)으로 빌드 없이도 소비 가능하게 함.

## 배포 산출물 형태

**소스 배포 + 설치 시점 빌드**입니다. 패키지에는 아래가 포함됩니다(`files` 기준).

| 형태 | 설명 | 사용처 |
| --- | --- | --- |
| `src/lib/` | TS/JSX 원본 + `tokens.css` | 소비 시스템 Tailwind 가 클래스 스캔, 토큰 import |
| `dist/` | ESM/CJS 번들 + 타입 선언 — **커밋하지 않고** 설치 시 `prepare`(`yarn build`)가 생성 | 컴포넌트 import |
| `tailwind.preset.js` | 토큰 → Tailwind 유틸리티 매핑 프리셋 | 소비 시스템 `tailwind.config` |

진입점 정의는 루트 `package.json` 이 원본입니다:

```jsonc
{
  "exports": {
    ".": { "types": "./dist/index.d.ts", "import": "./dist/index.mjs", "require": "./dist/index.cjs" },
    "./styles": "./src/lib/tokens.css",   // 디자인 토큰(CSS 변수), "./tokens" 도 동일
    "./tailwind-preset": "./tailwind.preset.js"
  },
  "peerDependencies": { "react": ">=18", "react-dom": ">=18" }
}
```

하네스(`src/pages`, `src/api`, `src/providers`, `src/App.tsx` 등)는 패키지에 **포함되지 않습니다**
— 소비 시스템이 최초 1회 복사해 소유하는 템플릿입니다. ([03. 시작하기](03-getting-started.md))

## 보안 · 분리 고려사항

- **비밀정보 금지**: 라이브러리에는 어떤 자격증명·내부 호스트명·토큰도 포함하지 않습니다(순수 UI만).
- **외부 호출 없음**: 컴포넌트는 외부 도메인으로의 네트워크 요청(폰트/아이콘 CDN 포함)을 하지 않습니다. 폰트·아이콘은 **번들에 포함**합니다.
- **텔레메트리 없음**: 격리 환경 특성상 분석/추적 코드는 넣지 않습니다.
- **공급망 최소화**: 의존성 추가는 리뷰에서 보안·라이선스·격리 재현성을 함께 검토합니다.
